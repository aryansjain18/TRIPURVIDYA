  # Representation of Sets

A set can be represented in two standard methods. These methods help us clearly describe the elements of a set.
*The two main ways to represent a set are:
Roster (Tabular) Form
Set-Builder Form

1. Roster Form (Tabular Form)

# Definition
In Roster Form, all the elements of a set are listed explicitly inside curly brackets { }.
Example
A = {2, 4, 6, 8}
This represents a set of even numbers less than 10.
More Examples
Set of vowels:
V = {a, e, i, o, u}
Set of first five natural numbers:
N = {1, 2, 3, 4, 5}
Important Points (Roster Form)
Elements are separated by commas
Repetition of elements is not allowed
Order of elements does not matter
# Example:
{1, 2, 3} = {3, 2, 1}
When to Use Roster Form
When the number of elements is small
When elements can be easily listed
2. Set-Builder Form

# Definition
In Set-Builder Form, a set is represented by describing a rule or condition that its elements satisfy.
General Notation
A = { x | x satisfies a given condition }
Here, the symbol | means “such that”.

# Example
A = { x | x is a natural number less than 6 }
This represents the same set:
A = {1, 2, 3, 4, 5}


**More Examples
Even natural numbers:
E = { x | x is an even natural number }

Multiples of 3 less than 15:
M = { x | x < 15 and x is a multiple of 3 }

# Important Points (Set-Builder Form)
* Elements are not written one by one
* A condition defines all elements
* Very useful for large or infinite sets
