Python Syntax

# What is Python Syntax?

Python syntax refers to the rules and structure that define how Python programs are written and executed.
Python syntax is known for being:
Simple
Clean
Easy to read
Because of this, Python is very beginner-friendly.

# Statements in Python

A statement is an instruction given to Python to perform a specific task.
Important points:
Python executes statements from top to bottom
Each statement represents one action
Multiple statements together form a program

# Indentation in Python ⭐ (Most Important)

Unlike many other languages, Python uses indentation instead of curly braces to define blocks of code.
Key points:
Indentation means leading spaces at the beginning of a line
It shows which statements belong together
Proper indentation is mandatory in Python
Incorrect indentation can cause errors or wrong output.

# Case Sensitivity

Python is a case-sensitive language.
This means:
Uppercase and lowercase letters are treated differently
Names must be written consistently throughout the program

# Comments in Python

Comments are used to:
Explain the logic of the program
Improve readability
Make maintenance easier
Important points:
Comments are ignored during execution
They do not affect program output

# Whitespace in Python

Whitespace includes:
Spaces
Tabs
Blank lines
Python uses whitespace to:
Improve readability
Define code blocks through indentation
Extra whitespace inside a line is generally ignored.

# Identifiers in Python

Identifiers are names given to:
Variables
Functions
Classes
Rules for identifiers:
Must start with a letter or underscore
Cannot start with a number
Cannot contain spaces
Cannot be a reserved keyword

# Keywords in Python

Keywords are reserved words with special meanings.
Important facts:
Keywords control program logic
They cannot be used as identifiers
Python keywords are written in lowercase

# Python Execution Style

Python is an interpreted language, which means:
Code is executed line by line
Errors stop execution at the point where they occur
No separate compilation step is required
